# Deployment Checklist

## Before Deployment

- confirm the target repository and branch
- note the files or modules being changed
- capture a rollback idea
- identify any environment or secret dependencies
- record the verification plan

## After Deployment

- verify the intended surface is healthy
- check the primary user-facing flow
- note any residual risk or follow-up cleanup
- promote lasting lessons into `context/` or `DECISIONS.md`
